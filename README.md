# Микросервисный банковский проект

Проект реализует банковскую систему на основе микросервисной архитектуры.
Каждый сервис представляет собой отдельное Java-приложение, изолированное и развертываемое независимо.

# Архитектура сервисов

### API Gateway

**Api-Gateway** — внешний маршрутизатор запросов.
Принимает входящие HTTP-запросы и проксирует их в соответствующие микросервисы.

### Account Service

**Account-service** — управление аккаунтами и их балансами.

### Cash Service

**Cash-service** — операции пополнения и списания средств.

### Transfer Service

**Transfer-service** — переводы средств между счетами.

### Notification Service

**Notification-service** — сервис уведомлений.
В текущей реализации уведомления логируются в консоль.

### Front Service

**Front-service** — отдельное клиентское приложение (не является частью микросервисной архитектуры), взаимодействующее с **API Gateway**.

# API документация

В папке docs находятся **OpenAPI-спецификации** в формате JSON для:

- account-service

- notification-service

На основе этих спецификаций можно автоматически генерировать Java-клиенты для взаимодействия с сервисами.

# Service Discovery и конфигурация

Для регистрации и обнаружения сервисов используется **Consul**. Конфигурация сервисов хранится в Consul в формате YAML.

# Аутентификация и авторизация

В качестве сервера авторизации используется

    Keycloak

# Межсервисное взаимодействие

Микросервисы взаимодействуют напрямую друг с другом с использованием:

    OAuth 2.0 Grant Type: Client Credentials

# Миграции базы данных

Для управления схемой базы данных используется **Liquibase**.

# Контейнеризация

Все сервисы упакованы в Docker-контейнеры и запускаются через **Docker Compose**.
